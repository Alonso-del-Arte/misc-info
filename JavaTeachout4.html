Conflicting Scales

Floating point can represent numbers...

  • ...as large as 1.8 × 10308 (the visible universe is believed to contain just 1080 atoms)
  • ...as tiny as 2.2 × 10−308 (tinier than the diameter of an electron in inches)

But what about a number like

  • 1.8 × 10308 + 2.2 × 10−308 ?

The smaller number vanishes to 0.

Error Accumulation

A single floating point operation may have an acceptable degree of error, but several floating point operations building upon each other might accumulate errors to an unacceptable degree.

Error Accumulation

 > 0.3 * 10000
<· 3000.0

...but set num = 0 and repeat num = num + (0.1 + 0.2) ten thousand times:

<· 3000.0000000003583

The extra 0.0000000003583 is the accumulated error of repeatedly adding 0.1 and 0.2.

Floating Point Weirdness

Takeaways

  • Floating point is unsuitable for money arithmetic.
  • Avoid repeated addition or subtraction, prefer multiplication whenever possible.
  • Accept that floating point arithmetic is not always precise, consider if it is or isn't good enough for your purpose.

Array Basics

Array Basics

An array is a group of elements of a specified type. An array is of fixed size.

Advantages of Arrays

  • Easy to declare
  • Easy to initialize
  • Fast access to elements
  • Backwards compatibility to JDK 1.0

Disadvantages of Arrays

  • Can't grow array in place
  • Inserting at the beginning requires copying and moving all of the existing elements you want to keep
  • Inserting in the middle requires copying and moving the existing elements and the desired position and all later elements

Array Declaration Syntax

Here's one way to declare an array:

type[] identifier = new type[positive integer];

For example:

int[] someNumbers = new int[3];

There is also a slightly different syntax inherited from C/C++.

Array Declaration and Initialization Syntax

If the array's to have just a few elements, you can use an "array literal"

type[] identifier = { literals or variables separated by commas };

For example:

int[] someNumbers = {1, 2, 3, 4, 7};

Java Array Initialization

If not immediately declared with specific initial values, the elements of an array will be initialized to zero, false or null, as applicable.

    public static void main(String[] args) {
        int[] myIntArray = new int[3];
        System.out.println(myIntArray[0]);
        System.out.println(myIntArray[1]);
        System.out.println(myIntArray[2]);
        myIntArray[0] = 3;
        myIntArray[1] = 4;
        myIntArray[2] = 6;
        System.out.println(myIntArray[0]);
        System.out.println(myIntArray[1]);
        System.out.println(myIntArray[2]);
    }

This should print out three zeroes before the array elements are expressly assigned, and the numbers 3, 4, 6 afterwards, regardless of which system with the Java Runtime Environment we run this on.

Java Array Bounds Enforcement

int[] someNumbers = new int[3];
System.out.println(someNumbers[-10]);

Length Property of an Array

Arrays have a length property that can be accessed to find out how many elements the array has.

       int[] someNumbers = new int[3];
       System.out.println(someNumbers.length); // 3
       double[] moreNumbers = {1.25, Math.sqrt(2), 3.8, 
                               Math.random(), 1e85};
       System.out.println(moreNumbers.length); // 5
       System.out.println(args.length); // Probably 0

Number of Command Line Arguments

The args array passed to public static void main() contains the command line arguments (if any) that were passed in from the command line.

args.length will be 0 if there were no command line arguments, a positive integer if there was at least one command line argument.

Collections

Quick Look at Three Types of Collections

  • Lists
  • Sets
  • Queues

List Properties

  • A list has a particular order
  • A list can grow as needed
  • A list can have duplicate elements

List Operations

  • Create an empty list, or create a list from an array or from another collection
  • Add an element (optional: specify position)
  • Retrieve the element at an arbitrary position
  • Remove an element
  • Reorder the list
  • Query size of the list
  • Copy the list to an array or to another collection
  • Clear out the list

Examples of Lists

  • A list of integers: 1, 1, 2, 1, 3, 1, 2, 4, 1, 5, 1, 2, 3, 6, 1, 7, 1, 2, 4, 8, 1, 3, 9, 1, 2, 5, 10, 1, 11, 1, 2, 3, 4, 6, 12, 1, 13, 1, 2, 7, 14, 1, 3, 5, 15, 1, 2, 4, 8, 16, 1, 17, 1, 2, 3, 6, 9, 18, 1, 19, 1, 2, 4, 5, 10, 20, 1, 3, 7, 21, 1, 2, 11, 22, 1, 23, 1, 2, 3, 4, 6, 8, 12, 24, 1, 5, 25, 1, 2, 13, 26, 1, 3, 9, 27, 1, 2, 4, 7, 14, 28
  • A list of Strings: "Dasher", "Dancer", "Prancer", "Vixen", "Comet", "Cupid", "Donner", "Blitzen", "Rudolph"

Set Properties

  • A set has no particular order
  • A set can grow as needed
  • A set can't have duplicate elements

Set Operations

  • Create an empty set, or create a set from an array or from another collection
  • Add an element
  • Retrieve the element pointed to by an iterator
  • Remove an element
  • Query size of the set
  • Get the union, intersection or complement of the set with another set
  • Copy the set to an array or to another collection
  • Clear out the set

Set Operations

NOTE: You can try to add to a set an element that the set already contains. This should not cause any exception, but there should be some indication that the duplicate add was rejected.

Examples of Sets

  • A set of integers: 0, 1, 4, 9, 3, 12, 10. This may or may not be the order in which the integers are stored internally.
  • A set of playing cards in a poker hand: 7♠, 8♠, 9♠, 10♠, Q♥. If the cards were shuffled properly, the player probably got them in a different order. And if there are any duplicate cards, it probably means someone's cheating.

Queue Properties

  • A queue has a particular order
  • A queue can grow as needed
  • Elements can be added at the beginning or at the end, but not in the middle

Queue Operations

  • Create an empty queue
  • Create a queue from an array or from another collection
  • Add an element to the queue
  • Remove an element
  • Query size of the queue
  • Copy the queue to an array
  • Copy the queue to another collection
  • Clear out the queue
